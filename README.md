# Skypro Kanban

Учебное Kanban-приложение на React с авторизацией, работой с задачами через API и поддержкой тем.

## Основной функционал

- Авторизация и регистрация через API (`/user`, `/user/login`), токен хранится в `localStorage`.
- Защищённые роуты (`/`, `/card/:id`, `/new-card`, `/exit`) только для авторизованных пользователей.
- Получение/создание/редактирование/удаление задач через API `/kanban`.
- Отображение статусов задач по колонкам доски.
- Обработка ошибок API и сетевых сбоев, показ понятных сообщений пользователю.

## Темы (ThemeProvider)

- Реализована смена темы с помощью `styled-components` `ThemeProvider` и собственного контекста `ThemeContext`.
- Доступные темы: **светлая** и **тёмная** (см. `src/theme.js`).
- Выбранная тема сохраняется в `localStorage` и восстанавливается при перезагрузке.
- Переключатель темы находится в попапе пользователя в хедере (чекбокс «Темная тема»).
- Глобальные стили (`GlobalStyles`) читают текущую тему и меняют фон и основной цвет текста.

## Оповещения (React-Toastify)

Для продвинутых оповещений используется библиотека **React-Toastify**.

Где используются тосты:

- **Авторизация** (`LoginPage`):
  - Успех: «Вы успешно вошли в аккаунт».
  - Ошибка: отображается текст ошибки от сервера либо сообщение по умолчанию.
- **Регистрация** (`RegisterPage`):
  - Успех: «Регистрация прошла успешно, теперь можно войти».
  - Ошибка: текст ошибки от сервера или сообщение по умолчанию.
- **Задачи**:
  - Создание задачи (`PopNewCard`): тост об успешном создании или ошибке.
  - Редактирование задачи (`PopBrowse`): тост об успешном сохранении или ошибке.
  - Удаление задачи (`PopBrowse`): тост об успешном удалении или ошибке.

`ToastContainer` настроен в `ThemeContext`, тема тостов (`light`/`dark`) автоматически подстраивается под текущую тему приложения.

## Дополнительные проверки и UX

- Валидация форм логина и регистрации:
  - Поля `email`, `password` и `name` (на регистрации) очищаются от пробелов по краям (`trim()`).
  - Нельзя отправить пустые поля или поля, состоящие только из пробелов — показывается красное сообщение.
- Валидация задач:
  - Нельзя создать/сохранить задачу с пустым заголовком (после `trim()`).
- Загрузка:
  - Добавлен анимированный спиннер в компоненте `Loader` вместо простой надписи.
- Пустой список задач:
  - При отсутствии задач вместо ошибки показывается надпись «Новых задач нет».
- Страница 404:
  - Стилизована в стиле остальных страниц (фон, шрифт, цвета).
  - Есть кнопка-ссылка «На главную», ведущая на `/`.
